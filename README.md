# 抖喵游戏屋

一个使用 React + TypeScript + Node.js + Socket.IO 搭建的实时联机小游戏项目。当前主玩法是锤子剪刀布，包含大厅、房间、聊天、观战、Bot、排位、惩罚模式、名字争夺战、后台文本配置工具等功能。

## 本地运行

```bash
npm install
npm run dev
```

- 前端默认地址：`http://127.0.0.1:5173`
- 后端默认地址：`http://127.0.0.1:3000`

## 构建检查

每次修改后建议运行：

```bash
npm run build
npm run server:build
```

## 后台与配置

- 后台入口：`/admin` 或 `#admin`
- 仓库默认不保存后台口令。
- 部署时建议用环境变量设置后台口令：

```bash
ADMIN_PASSWORD=你的后台口令
```

配置文件说明：

- `config/default.json`：默认配置，会提交到 Git。
- `config/active.json`：服务器运行时实际配置，已被 `.gitignore` 忽略，不会提交。
- `work/uploads`：玩家证明图片和后台图片上传目录，已被 `.gitignore` 忽略。

## 最近更新记录

### 2026-06-14

- 明确极限模式关闭条件：排位分必须大于 0 才能关闭，0 分不能关闭。
- 新增黑白棋 V1：创建房间可选择黑白棋，支持真人 1v1、观战、聊天、断线恢复、合法落子、自动跳过、终局胜负、再来一局和对局记录；黑白棋暂不接入 Bot、惩罚和白给模式。
- 黑白棋接入普通排位：创建黑白棋房可选择 1/2/5/10 分/子，每次落子翻掉对方棋子时立即按翻子数 × 每子分加减，终局只记录胜负不重复结算，并进入现有在线积分榜和全局排行榜。
- 黑白棋排位补充倍率和极限模式：倍率会放大每子实时结算分，极限模式会在每次翻子结算时套用现有极限折扣；白给模式仍不对黑白棋开放。
- 黑白棋断线判负细化：对局中玩家断线仍先保留 30 秒静默 + 60 秒倒计时，超时后离线方判负，对局记录保存当前棋盘比分和已实时结算的排位变化，不再额外重复扣分。
- 后台管理操作新增黑白棋强制处理：管理员可重开黑白棋、判黑方胜、判白方胜或判平局；强制结束只写胜负和记录，不额外补扣排位分。
- 全局排行榜新增黑白棋分类：可查看黑白棋胜场榜、吃子榜和被吃榜，每行显示胜负平、吃子、被吃和净吃子。

### 2026-06-13

- 修复房间聊天记录：加入或创建房间时立即加载服务器返回的完整房间快照，不再需要刷新一次才看到历史聊天。
- 优化白给自救板：卡片更紧凑，去掉重复白给值显示，投票额度改为按点击者每小时分别计算点赞 3 次、倒赞 10 次，并显示剩余额度和刷新倒计时。
- 修复白给自救板标题数量标签过高的问题，让人数/条数徽章保持小尺寸显示，手机端也不会撑开标题区。
- 房间聊天增加切换：可在房间内查看“本房间”和“大厅聊天室”，大厅内容沿用留言板并在房间内只读。
- 新增排位倍率模式：玩家可花费 200 排位积分解锁倍率房，支持 x2 / x5 / x10，结算按基础赌分乘倍率计算，并在进房前提示风险。
- 新增全局排行榜弹窗：顶部增加排行榜入口，支持正分榜、负分榜、名字争夺战榜和白给榜，每类最多显示 50 名并标记在线状态。
- 新增极限模式：个人设置可开启极限模式，开启时排位分归零但保留胜负平和惩罚次数；极限排位房禁止倍率/Bot，支持分段加减分折扣、整点扣分和连胜风险。
- 排行榜补充极限正分榜和极限负分榜，方便单独查看开启极限模式的高分和低分玩家。
- 合并玩家名牌状态标识：名争、极限、极限名争统一显示为一个小标签，避免名字前后重复堆标识。
- 优化失名者显示：失名状态下不再额外堆状态标签，极限失名玩家直接显示为“极 + 代号/玩家改名”。
- 优化大厅房间排位标签：显示为“排位 20 分”，倍率房合并显示为“排位 20 分 ×10”。
- 修复对战比分结算文字溢出：长名字胜利文案会自动换行，手机端和电脑端都不会把比分卡片顶出边界。
- 修复对局记录长名字布局：玩家名和结果会在卡片内正常换行，手机端改为上下排列，避免被挤成竖条。
- 调整极限连胜风险：连胜触发时改为按概率额外扣 333 分，不再直接变成 -999 分。
- 调整极限/普通排位进房规则：排位类型不匹配时可以进入房间观战，但不能自动坐下或手动上桌。
- 修复页面开着但实际断联的问题：Socket 重连成功后会自动用本地凭证重新同步玩家、房间和座位状态。
- 对局记录电脑端也改为上下排列，彻底避免长名字在横向三列里被挤成竖条。
- 名字争夺战失格者离线后会继续在名单保留 30 分钟，并显示离线保留倒计时，方便其他玩家仍可抢先改名。

### 2026-06-12

- 新增白给模式：个人设置可开启白给模式，真人对战出拳区显示“白给”按钮；Bot 对战不显示也不会触发白给。
- 新增白给自救板：玩家可提交 12 小时自罚宣言，其他玩家可点赞降低白给值或倒赞增加白给值，后台可修改自救板文案。
- 修复白给模式细节：强制白给结算文案更明确，玩家名牌公开显示白给值，已上板玩家不再重复显示自救提交框。
- 调整白给结算：白给现在是正式出拳状态，点击后会锁定等待结算；双方白给算双输，排位双方扣分，惩罚房双方受罚。
- 修复排位断线判负：断线倒计时开始时锁定胜负对象，对手之后离开房间或下桌也会正常加减排位分。
- 调整大厅布局：名字争夺战失格者面板移动到大厅房间列表下方，并预留“白给内容”开发中占位框；修复电脑端房间列表房间少时出现大块空白的问题。
- 降低实时同步压力：大厅广播合并，配置从高频大厅包拆出，房间更新不再重复发送完整聊天，对局记录默认只下发最近 20 条。
- 断线提示改成两段式：前 30 秒静默，不显示离线；30 秒后才显示 60 秒倒计时。
- 在线积分榜改为显示全部在线玩家。
- 优化大厅排版：房间列表、留言板、排行榜使用更稳定的滚动布局，减少页面抖动。
- 优化房间排版：AB 对战栏、房间玩家名单、聊天、对局记录在桌面和手机端更稳定。
- 修复一方已出拳但另一边为空时，新玩家无法坐到空位的问题。
- 移除仓库里的默认后台密码，后台口令改为部署时单独设置。
